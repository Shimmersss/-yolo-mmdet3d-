# 前后端启动/暂停方法（Windows + PowerShell）

适用目录：`E:\BD\MAVEN\毕设`

## 1. 后端（Spring Boot）
后端项目目录：`E:\BD\MAVEN\毕设\SOFT-rear`

powershell -NoProfile -ExecutionPolicy Bypass -File "E:\BD\MAVEN\毕设\status-all.ps1"

powershell -NoProfile -ExecutionPolicy Bypass -File "E:\BD\MAVEN\毕设\status-all.ps1" -Watch -IntervalSec 3

### 启动后端
> 当前项目依赖 Java 17。若系统默认 Java 还是 1.8，请使用下面这组命令（推荐）。

```powershell
$env:JAVA_HOME="C:\Program Files\Eclipse Adoptium\jdk-17.0.18.8-hotspot"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
Set-Location "E:\BD\MAVEN\毕设\SOFT-rear"
.\mvnw.cmd spring-boot:run
```

后端默认地址：`http://localhost:8080`

### 暂停后端
- 在后端运行的那个终端里按：`Ctrl + C`

### 验证后端是否运行
```powershell
Invoke-WebRequest -Uri http://localhost:8080/user -UseBasicParsing
```

## 2. 前端（Vite + Vue）
前端项目目录：`E:\BD\MAVEN\毕设\web`

### 首次安装依赖（或依赖变化后）
```powershell
Set-Location "E:\BD\MAVEN\毕设\web"
& "C:\Program Files\nodejs\npm.cmd" install
```

### 启动前端
> 如果终端里识别不到 `node`，用下面写法可直接启动。

```powershell
$env:Path="C:\Program Files\nodejs;$env:Path"
Set-Location "E:\BD\MAVEN\毕设\web"
& "C:\Program Files\nodejs\npm.cmd" run dev -- --host
```

前端默认地址：`http://localhost:3000`

### 暂停前端
- 在前端运行的那个终端里按：`Ctrl + C`

### 验证前端是否运行
```powershell
Invoke-WebRequest -Uri http://localhost:3000 -UseBasicParsing
```

---

## 3. 常用操作

### 重启后端
1. 在后端终端按 `Ctrl + C`
2. 重新执行“启动后端”命令

### 重启前端
1. 在前端终端按 `Ctrl + C`
2. 重新执行“启动前端”命令

### 同时启动前后端
- 开两个终端：
1. 终端 A 跑后端
2. 终端 B 跑前端

---

## 4. 常见问题

### 报错：Java 版本不兼容（class file version 61）
原因：使用了 Java 8 运行 Spring Boot 3。
处理：先执行文档中的 `JAVA_HOME` 和 `Path` 设置，再启动后端。

### 报错：`node` 不是内部或外部命令
原因：Node 已安装但当前终端 Path 未刷新。
处理：先执行文档中的 `$env:Path="C:\Program Files\nodejs;$env:Path"`，再启动前端。
